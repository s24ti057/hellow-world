# ソフトウェア工学 2026

## ソフトウェア工学とは
ソフトウェア工学は、単なるプログラミング技術ではなく、  
「ソフトウェアを社会に安全・継続的に提供するための体系的な方法論」である。  
現代のソフトウェアは、社会インフラ・ビジネス・日常生活のあらゆる領域を支えており、  
その開発には **品質（Quality）・コスト（Cost）・納期（Delivery）** の最適化が求められる。

ソフトウェアは物理的に劣化しないが、環境変化・技術進化・ユーザー要求の増大によって  
「悪化」するため、継続的な改善・再構築が不可欠である。  
そのため、要件定義・設計・開発プロセス・テスト・保守・バージョン管理など、  
開発の全工程を体系的に理解し、チームで共有することが重要となる。

また、ソフトウェア工学は「人と組織の学問」でもある。  
技術だけでなく、コミュニケーション、プロジェクトマネジメント、  
変化への対応力、再利用性、持続可能性といった観点が不可欠である。

## CI/CD
CI/CD は、ソフトウェア工学の「自動化」と「継続的改善」を体現する仕組みである。

### CI（Continuous Integration）
- コード変更を頻繁に統合し、  
- 自動テスト・静的解析（flake8 など）を実行し、  
- 問題を早期に発見するためのプロセス。

CI により、開発者は「変更が壊れていないか」を常に確認でき、  
品質を保ちながら高速に開発を進められる。

### CD（Continuous Delivery / Deployment）
- テスト済みのコードを自動で環境にデプロイし、  
- ユーザーへの価値提供を継続的に行う仕組み。

CD により、手動作業によるミスを減らし、  
「コードを書いたらすぐ公開される」という高速な開発サイクルが実現する。

GitHub Actions を使った CI/CD は、  
ソフトウェア工学で学んだ「効率化」「再現性」「品質保証」を実際に体験できる仕組みであり、  
現代の開発現場では必須の技術となっている。

## GitHub Pages
GitHub Pages は、GitHub リポジトリをそのまま Web サイトとして公開できる仕組みである。  
Jekyll によって Markdown を自動的に HTML に変換できるため、  
開発者は文章を書くことに集中できる。

さらに GitHub Actions と組み合わせることで、  
「push → 自動ビルド → 自動公開」という CI/CD パイプラインが構築できる。

これは、ソフトウェア工学で学んだ以下の概念を実際に体験できる場でもある：

- 自動化されたビルド  
- 再現性のあるデプロイ  
- バージョン管理と公開の連携  
- 開発プロセスの標準化  
- ドキュメントの継続的更新

GitHub Pages は単なる静的サイトではなく、  
ソフトウェア工学の思想をそのまま体現した「学習用の実践環境」と言える。

## ソフトウェア工学の全体的なまとめ
この講義では、ソフトウェア工学の基礎から実務に直結する内容まで、  
体系的かつ段階的に学ぶことができた。

### ● ソフトウェアの本質
- ソフトウェアは社会インフラであり、継続的な改善が必要  
- 劣化しないが悪化する  
- 変化への対応が必須

### ● ライフサイクルの理解
- ニーズ → 要件定義 → 設計 → 実装 → テスト → 運用 → 廃止  
- 各工程は独立しておらず、相互に影響し合う  
- 要件定義はプロジェクトの「バイブル」

### ● 開発プロセスの多様性
- ウォーターフォール：安定性・計画性  
- スパイラル：リスク駆動  
- 反復型：段階的開発  
- アジャイル：変化対応・高速提供  
- どれが正しいではなく、状況に応じて選択する

### ● 品質・コスト・納期（QCD）
- すべてを最大化することはできない  
- プロジェクトの性質に応じて優先度を決める  
- 品質管理にはテストだけでなく設計・分析が重要

### ● バージョン管理（git）
- 分散管理による強力な履歴管理  
- ブランチ運用による並行開発  
- GitHub による共同開発・レビュー・CI/CD 連携

### ● 自動化（CI/CD）
- 人的ミスを減らし、品質を保ちながら高速に開発  
- 現代の開発では必須のスキル

### ● 実務とのつながり
講義で扱った内容は、実際の開発現場でそのまま使われる。  
特に、要件定義・プロジェクトマネジメント・バージョン管理・CI/CD は、  
どの企業でも必須のスキルであり、ソフトウェア工学の中心的な要素である。

## まとめ
ソフトウェア工学は、単なる技術の学習ではなく、  
「ソフトウェアを社会に届けるための総合的な学問」である。  
変化し続ける環境で持続可能なソフトウェアを作るためには、  
技術・プロセス・チーム・自動化・品質管理を総合的に理解する必要がある。

この講義を通じて、ソフトウェア開発の全体像と、  
現代の開発に必要な考え方・姿勢・技術を体系的に学ぶことができた。
